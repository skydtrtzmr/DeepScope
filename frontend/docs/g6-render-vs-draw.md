# G6 v5: render() vs draw() — 增量添加节点必须用 render()

## 关键结论

**增量添加节点/边时，不能用 `draw()` 替代 `render()`。**

## 原因

根据 G6 v5 官方文档 (`docs/api/render.zh.md`)：

| | `render()` | `draw()` |
|---|---|---|
| 数据处理 | ✅ 处理数据更新 | ❌ 不处理 |
| 布局计算 | ✅ 执行布局算法 | ❌ 不执行 |
| 绘制 | ✅ 绘制到画布 | ✅ 绘制到画布 |

- **`draw()`** 仅负责将图形元素绘制到画布上，不涉及布局计算。适用场景：**仅修改了元素样式或状态，不需要重新计算位置**。
- **`render()`** 执行完整的渲染流程（数据处理 → 布局计算 → 绘制）。适用场景：**初始化图表、添加或删除了大量节点/边、需要重新计算所有元素位置**。

## 实测问题

使用 `addData()` + `draw()` 增量添加节点后：

1. **新节点无法拖动** — `DragElementForce` 依赖 d3-force simulation 实例，`render()` 在布局计算过程中会初始化/更新 simulation，而 `draw()` 跳过了这一步，新节点未被注册到 simulation 中
2. **新节点若手动设置初始位置，只能画出来但"没有骨架"** — `draw()` 只画了 Canvas 像素，没有更新力导向的内部数据结构

## 正确做法

增量添加节点应继续使用 `addData()` + `render()`。如需优化渲染速度，应从以下方向入手：

- 调高 `alphaDecay` 加速力导向收敛
- 在 `afterlayout` 或短延时后聚焦，避免布局未稳定时聚焦导致焦点偏移
- 给新增节点设置合理的初始位置（基于被探索节点位置散布），减少力导向收敛所需迭代次数
